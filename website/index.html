01 / Context first

Familiar workflows. Full context.

Keep history, working-tree changes, and branch actions together in one focused workspace. Move through everyday Git work while keeping the context behind each change in view.

02 / Repository workflow

A workflow built around your repository

01

Visual history

Follow branches and commits in a readable history, then open any change without leaving the flow.

02

Review-ready diff

Inspect staged and unstaged changes with the surrounding file and commit context close at hand.

03

Interactive rebase

Reorder, squash, edit, and drop commits through a guided interactive rebase workflow.

04

Safe branch actions

Create, switch, rename, and delete branches with the current repository state visible before you act.

03 / Under the hood

How Pluck is built

01

Familiar by design

Common Git actions follow interaction patterns familiar to developers who work in JetBrains IDEs.

02

Git underneath

Pluck uses the Git CLI for repository operations, so the source of truth stays with Git itself.

03

Focused on the work

The interface stays centered on understanding changes and completing the next repository action.

Install from the DMG

  1. 1Download the latest DMG from the latest release.
  2. 2Open the DMG and drag Pluck into the Applications folder.
  3. 3Pluck is not yet signed with an Apple Developer ID, so macOS Gatekeeper may mark it as “damaged” on first launch. Open Terminal and remove the quarantine attribute:
xattr -dr com.apple.quarantine /Applications/Pluck.app

Developer ID signing and notarization are planned. Once available, the xattr step will no longer be needed.
