# Pluck Modern UI Refresh Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild Pluck's frontend into a modern, dense git workbench inspired by CC Switch's clear control surfaces and the Codex app client's calm command workflow.

**Architecture:** Keep the existing Tauri/Rust backend, Pinia stores, invoke wrappers, and snapshot replacement model unchanged. The refresh is a frontend-only pass: introduce a sharper `gl-*` design system, add a lightweight icon library, and update the shell, panels, dialogs, banners, toasts, and status surfaces in small reviewable tasks.

**Tech Stack:** Vue 3 setup SFCs, Pinia, TailwindCSS 4, CSS variables in `src/style.css`, Tauri invoke wrappers, `lucide-vue-next` for icons.

## Global Constraints

- Work in `/Users/hzp/personal/pluck`.
- Do not change Rust backend behavior, git command behavior, `run_git`, snapshot assembly, or Tauri command contracts.
- Do not introduce a full UI component library; add only `lucide-vue-next` for consistent icons.
- Keep Pinia stores as setup stores.
- Keep all reusable visual primitives in `src/style.css` as `gl-*` classes.
- Do not use native `prompt`, `confirm`, or `alert`.
- Do not run full `pnpm build` after every small edit; run it once during final verification unless a type/config change requires earlier validation.
- Do not push to a remote branch unless the user explicitly requests or confirms.
- If commits are requested, write commit messages in Chinese and follow the repository convention, for example `feat(ui): 重构工作台界面`.
